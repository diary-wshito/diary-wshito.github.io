ファンタジー・ランドの保護地区をゆく Part 1

JavaScript の関数型プログラミング・ライブラリである Sanctuary.js のサーベイ．
November 13, 2018 on #sanctuary #fantasyland #javascript #functional programming

最近，JavaScript の関数型プログラミングにはまっているので，Fantasy Land を実装した Sanctuary.js の使い方をサーベイして行きたいと思います．今回は基本中の基本，Functor から．間違いなどありましたらコメントお願いします．

Functor とは何か

Sanctuary が提供する型は，ここの README.md にまとめられています．様々な関数が Functor を引数に取るので，まずは Functor から見ていきましょう．

色々なサイトで Functor とは何ぞやという分かりやすい説明があるので（例えば「箱で考えるFunctor、ApplicativeそしてMonad」やProfessor Frisby Introduces Composable Functional JavaScript），詳しい解説はそちらに譲ります．ここではあらゆるデータをラッピングして格納するラッパー（もしくは箱）ということだけ理解していれば十分です．そのラッパーは map メソッドを提供します．

ラッパーである Functor は内部にデータだけでなく関数も保持できます．（というより，関数型プログラミング言語の関数は第1級オブジェクトなので，関数もデータと変わりありません．従って，データをラッピングする Functor には関数も格納できます．）

何が Functor になり得るの？

map メソッドを提供している型が Functor です．Sanctuary の型システムを見ると多くの型が Functor の派生型として実装されていることがわかります．これら以外にも配列とオブジェクトも Sanctuary では Functor の派生型として扱われます．

Functor か否かは Functor.test() メソッドを使って調べることができます．

$ node
> let Z = require('sanctuary-type-classes')  // 型の定義
> let S = require('sanctuary')               // Sanctuary本体

> Z.Functor.test(S.Maybe)          // Maybe型はFunctorのサブクラス
true
> Z.Functor.test(S.toMabye(5))     // Just型もFunctor
true
> Z.Functor.test(S.Just)
true
> Z.Functor.test(S.toMabye(null))  // Nothing型もFunctor
true
> Z.Functor.test(S.Nothing)
true

> Z.Functor.test({})   // オブジェクトもFunctor
true

> Z.Functor.test([])   // 配列もFunctor
true

> Z.Functor.test(5)    // ただの値はFunctorではない
false

map メソッドを見てみよう

Functor の定義

  1. map メソッドを持つこと．

  2. u.map(a ⇒ a) は自分自信 u と同値であること．

  3. u.map(x ⇒ f(g(x))) は u.map(g).map(f) と同値であること．

コード 1. map の型定義
map :: Functor f => f a ~> (a -> b) -> f b

型定義を読めないと，Sancutary で迷子になってしまうので，まずはコード 1の型定義を一つずつ見ていきましょう．

  • Functor f ⇒ の部分で f に Functor 型の制約を置いています．

  • f a ~> の部分は，map メソッドが帰属するオブジェクトの型を表しています．つまりファンクター f は内部に値として a という任意の型を持つことを表しています．

  • (a → b) は map の第 1 引数が Type a から Type b への関数であることを表しています．

  • 最後の f b は map の戻り値の型を表しています．Type b を格納する Functor 型が帰ります．

例として，配列への map 呼び出しを見てみましょう．

コード 2. 配列への map
[4].map(Math.sqrt)
//=> [ 2 ]

まず，map 関数が呼び出されるオブジェクトの型は配列で，配列の中は number 型なので，f a ~> の部分は f が配列型で，a が number 型になります．次は，map の第 1 引数ですが，Math.sqrt は number → number の関数で，コード 1 の (a → b) 部分に対応しています．したがって b も number 型になります．コード 2 の戻り値は [2] で number を内部にもつ配列なので，確かに f a を満たしています．

Functor への map 関数の提供は，オブジェクト・メソッドを呼び出す以外に，独立した関数呼び出しによっても実現できます．独立した map 関数はカリー化されているので，下のように第 1 引数から順に関数呼び出しする必要があります．

S.map(Math.sqrt)([4])  // fantasy-land/mapのカリー化バージョン
//=> [ 2 ]

もう少し，例を見てみましょう．

[1, 2, 3, 4, 5].map(Math.sqrt)
//=> [ 1, 1.4142135623730951, 1.7320508075688772, 2, 2.23606797749979 ]

S.map(Math.sqrt)([1, 2, 3, 4, 5])  // fantasy-land/mapのカリー化バージョン
//=> [ 1, 1.4142135623730951, 1.7320508075688772, 2, 2.23606797749979 ]

次に，オブジェクトも Functor として扱えるので，オブジェクトに対しても map を実行してみましょう．

Z.Functor.test({a: " 1 ", b: " 2 "});
//=> true

{a: " 1 ", b: " 2 "}.map(Math.sqrt)    // Object.prototype.map はありません！
//=> エラー！

S.map(Math.sqrt)({a: " 1 ", b: " 2 "}) // こちらを使いましょう．
//=> { a: 1, b: 1.4142135623730951 }

上の例を見て分かる通り，オブジェクトには map メソッドがないので，独立した map 関数を使う方法しかありません．

当然，Functor の定義を満たしてないやんか！と突っ込みたくなりますが，まぁ，メソッドが提供されてなくとも map 関数は適用できるということで大目に見ましょう．似たようなケースとして，StackOverflow では Contravariant Functor のインスタンスに contramap メソッドが定義されていないという指摘がありましたが，皆さん，独立した関数で OK やんかというお答えのようです．

関数型プログラミングでは継続よりもポイントフリー・スタイルで書くことが多いでしょうから，独立した map 関数が使えれば十分ですね．
